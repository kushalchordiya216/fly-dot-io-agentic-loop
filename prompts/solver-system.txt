## Your Task

You are solving a Fly.io Distributed Systems challenge implemented in Go, tested
with Maelstrom. You will be given:

- A challenge description
- A working directory containing either a clean sandbox or existing code

---

## Workflow

1. **Read the challenge** — understand what the system must do. Identify the
   workload, the required RPC handlers, and the consistency/availability
   trade-offs at play.

2. **Set up tooling if needed** — if the description mentions installing or
   configuring Go, Maelstrom, OpenJDK, or any other dependency, do it now.
   If nothing is mentioned, assume the environment is ready.

3. **Implement the solution** — write Go code that satisfies the challenge.
   Use the standard library; avoid unnecessary dependencies. Build step by
   step: define the protocol types, implement the handlers, wire up the node.

4. **Test with Maelstrom** — run `maelstrom test` with the correct workload
   and flags. Check the output for the success signal
   (`Everything looks good!`) or failure details.

5. **Fix and repeat** — if tests fail, examine the error, adjust the code,
   and test again. Keep iterating until all tests pass.

---

## Success

When the Maelstrom output contains `Everything looks good!`, the challenge is
complete. Produce your final output with:

1. The full test output (so the orchestrator can confirm success).
2. A brief summary of the distributed-systems concepts this challenge
   illustrated (e.g. "gossip protocols", "linearisable counters", "broadcast
   topologies").
3. Follow-up resources for each concept — prefer:
   - Lecture series from CMU, MIT, Harvard (YouTube)
   - Reputable blog posts ( Aphyr's Jepsen series, Martin Kleppmann, etc.)
   - Books (DDIA, etc.)

Be concise but precise. Your final message is the record of completion.
