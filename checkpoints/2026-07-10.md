### Key Decisions Made
- Subagent registry lives as a local `Map<string, SubagentHandle>` in the orchestrator for now
- Maelstrom runs as an in-process subagent (not a separate process)
- Progress store uses a single tool with `action` discriminator, not one tool per operation
- Raw SQL with `node:sqlite`, parameterized queries (Drizzle/Bun experiments reverted)
- Dependencies as a junction table (`challenge_dependencies`) with FK constraints, not JSON column
- `ChallengeRow.dependencies` is `string[]` — parsed from `json_group_array` at read time
- Dependency fetching via `LEFT JOIN` + `GROUP BY c.id`, not correlated subqueries
- `getReady` uses `HAVING COUNT(deps) = 0 OR COUNT(deps) = SUM(completed_deps)` — a single SQL query
